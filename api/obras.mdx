---
title: "Obras"
description: "Endpoints for managing construction projects (obras) and their sub-resources."
---

Obras are the central entity in Sintesis. Each obra belongs to a tenant and represents a construction contract with financial and scheduling data.

---

## List obras

<ParamField path="GET /api/obras" type="string" />

Returns all obras for the authenticated user's active tenant. Supports pagination, sorting, full-text search, and range filters.

### Query parameters

<ParamField query="page" type="number" default="1">
  Page number (1-indexed). Enables paginated mode when provided alongside `limit`.
</ParamField>

<ParamField query="limit" type="number" default="50">
  Results per page. Clamped between `1` and `500`.
</ParamField>

<ParamField query="orderBy" type="string" default="n">
  Column to sort by. Allowed values: `n`, `designacion_y_ubicacion`, `sup_de_obra_m2`, `entidad_contratante`, `mes_basico_de_contrato`, `iniciacion`, `contrato_mas_ampliaciones`, `certificado_a_la_fecha`, `saldo_a_certificar`, `segun_contrato`, `prorrogas_acordadas`, `plazo_total`, `plazo_transc`, `porcentaje`, `updated_at`.
</ParamField>

<ParamField query="orderDir" type="string" default="asc">
  Sort direction: `asc` or `desc`.
</ParamField>

<ParamField query="status" type="string">
  Filter by completion status. Accepted values: `completed` (porcentaje = 100) or `in-process` (porcentaje < 100).
</ParamField>

<ParamField query="q" type="string">
  Full-text search across `designacion_y_ubicacion`, `entidad_contratante`, `mes_basico_de_contrato`, `iniciacion`, and numeric fields.
</ParamField>

<ParamField query="supMin" type="number">
  Minimum surface area (m²).
</ParamField>

<ParamField query="supMax" type="number">
  Maximum surface area (m²).
</ParamField>

<ParamField query="porcentajeMin" type="number">
  Minimum completion percentage.
</ParamField>

<ParamField query="porcentajeMax" type="number">
  Maximum completion percentage.
</ParamField>

<ParamField query="entidad" type="string[]">
  Filter by one or more contracting entities. Repeat the parameter for multiple values: `?entidad=MUNICIPIO&entidad=PROVINCIA`.
</ParamField>

<ParamField query="mesYear" type="string">
  Filter `mes_basico_de_contrato` by year substring.
</ParamField>

<ParamField query="iniYear" type="string">
  Filter `iniciacion` by year substring.
</ParamField>

<Warning>
Numeric range filters are also available for `contrato_mas_ampliaciones` (`cmaMin`/`cmaMax`), `certificado_a_la_fecha` (`cafMin`/`cafMax`), `saldo_a_certificar` (`sacMin`/`sacMax`), `segun_contrato` (`scMin`/`scMax`), `prorrogas_acordadas` (`paMin`/`paMax`), `plazo_total` (`ptMin`/`ptMax`), and `plazo_transc` (`ptrMin`/`ptrMax`).
</Warning>

## Bulk upsert obras

<ParamField path="PUT /api/obras" type="string" />

Replaces the entire obras list for the tenant. Records present in the current tenant but absent from the payload are deleted. New obras automatically have tenant-level defaults applied (folders, tablas).

When an obra transitions from `porcentaje < 100` to `porcentaje = 100`, an `obra.completed` workflow event is emitted and any configured flujo actions are executed.

### Request body

<ParamField body="detalleObras" type="Obra[]" required>
  Full array of obras to persist. Each item must conform to the obra schema.
  <Expandable title="Obra input fields">
    <ParamField body="n" type="number" required>Obra number — unique per tenant.</ParamField>
    <ParamField body="designacionYUbicacion" type="string" required>Name and location.</ParamField>
    <ParamField body="supDeObraM2" type="number" required>Surface area (m²).</ParamField>
    <ParamField body="entidadContratante" type="string" required>Contracting entity.</ParamField>
    <ParamField body="mesBasicoDeContrato" type="string" required>Base contract month.</ParamField>
    <ParamField body="iniciacion" type="string" required>Start date.</ParamField>
    <ParamField body="contratoMasAmpliaciones" type="number" required>Contract value including extensions.</ParamField>
    <ParamField body="certificadoALaFecha" type="number" required>Amount certified to date.</ParamField>
    <ParamField body="saldoACertificar" type="number" required>Remaining amount to certify.</ParamField>
    <ParamField body="segunContrato" type="number" required>Value per original contract.</ParamField>
    <ParamField body="prorrogasAcordadas" type="number" required>Agreed extensions.</ParamField>
    <ParamField body="plazoTotal" type="number" required>Total contract term (days).</ParamField>
    <ParamField body="plazoTransc" type="number" required>Elapsed term (days).</ParamField>
    <ParamField body="porcentaje" type="number" required>Completion percentage.</ParamField>
    <ParamField body="customData" type="object">Custom column values (keys must match tenant's enabled custom column IDs).</ParamField>
  </Expandable>
</ParamField>

### Response

<ResponseField name="ok" type="boolean" required>Always `true` on success.</ResponseField>

## Replace a single obra

<ParamField path="PUT /api/obras/{id}" type="string" />

Fully replaces all fields of an existing obra. Completion side-effects (workflow events, flujo actions) fire if `porcentaje` transitions to 100.

### Path parameters

<ParamField path="id" type="string" required>UUID of the obra.</ParamField>

### Request body

All obra fields (same schema as the bulk PUT body items, excluding `n` as identifier — the `id` path param is authoritative).

### Response

<ResponseField name="ok" type="boolean" required>Always `true` on success.</ResponseField>

---

## Partially update a single obra

<ParamField path="PATCH /api/obras/{id}" type="string" />

Updates only the supplied fields of an obra. Omitted fields are left unchanged. Completion side-effects fire if `porcentaje` transitions to or from 100.

### Path parameters

<ParamField path="id" type="string" required>UUID of the obra.</ParamField>

### Request body

Any subset of obra fields.

### Response

<ResponseField name="ok" type="boolean" required>Always `true` on success.</ResponseField>

      --cookie 'sb-access-token=<session>' \
      --header 'Content-Type: application/json' \
      --data '{ "porcentaje": 100 }'

## Bulk patch obras

<ParamField path="PATCH /api/obras/bulk" type="string" />

Upserts multiple obras in a single request. Unlike `PUT /api/obras`, this endpoint does **not** delete obras absent from the payload. Defaults are applied to newly created obras.

### Request body

<ParamField body="updates" type="Obra[]" required>
  Array of obra objects to upsert. Each item must include either `id` (UUID) or a positive `n` value as identifier. Minimum 1 item.
</ParamField>

### Response

<ResponseField name="ok" type="boolean" required>Always `true` on success.</ResponseField>
<ResponseField name="count" type="number" required>Number of obras processed.</ResponseField>
<ResponseField name="supportsConfigColumns" type="boolean" required>Whether `on_finish_*` columns are available in this tenant's schema.</ResponseField>
