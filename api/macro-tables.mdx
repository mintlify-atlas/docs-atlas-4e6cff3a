---
title: "Macro Tables"
description: "Endpoints for managing macro tables — cross-obra aggregation views with configurable columns and editable custom values."
---

Macro tables aggregate rows from multiple `obra_tablas` across different obras into a single view. Each macro table has a set of columns (source, custom, or computed) and an optional set of source obra_tablas.

Sources can be selected in two modes:
- **Explicit** — specific `obra_tablas` are linked to the macro table.
- **Template** (`sourceMode: "template"`) — sources are resolved dynamically based on a `defaultTablaId` pattern, matching tables across all obras in the tenant.

---

## Create a macro table

<ParamField path="POST /api/macro-tables" type="string" />

Creates a new macro table with sources and columns. At least one source and one column are required.

### Request body

<ParamField body="name" type="string" required>Display name. Must be unique within the tenant.</ParamField>
<ParamField body="description" type="string">Optional description.</ParamField>
<ParamField body="settings" type="object">Source selection settings.</ParamField>
<ParamField body="sources" type="SourceInput[]" required>
  Source obra_tablas. Must contain at least one item.
  <Expandable title="SourceInput fields">
    <ParamField body="obraTablaId" type="string" required>UUID of the obra_tabla to include as a source. Must belong to an obra in the tenant.</ParamField>
  </Expandable>
</ParamField>
<ParamField body="columns" type="ColumnInput[]" required>
  Column definitions. Must contain at least one item.
  <Expandable title="ColumnInput fields">
    <ParamField body="label" type="string" required>Display label.</ParamField>
    <ParamField body="columnType" type="string" default="source">`source`, `custom`, or `computed`.</ParamField>
    <ParamField body="sourceFieldKey" type="string | null">Required for `source` columns. Field key from source rows or `obra.<fieldName>` to reference obra fields.</ParamField>
    <ParamField body="dataType" type="string" default="text">Data type of the column.</ParamField>
    <ParamField body="config" type="object">Column configuration.</ParamField>
  </Expandable>
</ParamField>

### Response

<ResponseField name="macroTable" type="MacroTable" required>The newly created macro table with its sources and columns.</ResponseField>

<Tabs>
  <Tab title="Request">
    ```bash
    curl -X POST 'https://<domain>/api/macro-tables' \
      --cookie 'sb-access-token=<session>' \
      --header 'Content-Type: application/json' \
      --data '{
        "name": "Resumen de materiales",
        "sources": [{ "obraTablaId": "ot-uuid-..." }],
        "columns": [
          { "label": "Material", "columnType": "source", "sourceFieldKey": "material", "dataType": "text" },
          { "label": "Cantidad", "columnType": "source", "sourceFieldKey": "cantidad", "dataType": "number" }
        ]
      }'
    ```
  </Tab>
  <Tab title="Response">
    ```json
    {
      "macroTable": {
        "id": "mt-uuid-...",
        "name": "Resumen de materiales",
        "sources": [{ "id": "src-uuid-...", "obraTablaId": "ot-uuid-...", "position": 0 }],
        "columns": [
          { "id": "col-1-uuid", "label": "Material", "columnType": "source", "dataType": "text", "position": 0, "config": {} },
          { "id": "col-2-uuid", "label": "Cantidad", "columnType": "source", "dataType": "number", "position": 1, "config": {} }
        ]
      }
    }
    ```
  </Tab>
</Tabs>

## Update a macro table

<ParamField path="PATCH /api/macro-tables/{id}" type="string" />

Partially updates a macro table. Supplying `sources` or `columns` replaces those lists entirely (existing entries are deleted and new ones inserted). Fields not supplied are left unchanged.

<Warning>
When `columns` is supplied, existing custom values for deleted columns are also removed via cascade.
</Warning>

### Path parameters

<ParamField path="id" type="string" required>UUID of the macro table.</ParamField>

### Request body

<ParamField body="name" type="string">New display name.</ParamField>
<ParamField body="description" type="string">New description (empty string sets it to null).</ParamField>
<ParamField body="settings" type="object">Updated source selection settings.</ParamField>
<ParamField body="sources" type="SourceInput[]">Replacement source list. Replaces all existing sources.</ParamField>
<ParamField body="columns" type="ColumnInput[]">Replacement column list. Replaces all existing columns.</ParamField>

### Response

<ResponseField name="macroTable" type="MacroTable" required>The updated macro table with resolved sources and columns.</ResponseField>

## List rows

<ParamField path="GET /api/macro-tables/{id}/rows" type="string" />

Returns aggregated rows from all resolved source tablas, with source and custom column values mapped. Supports pagination, search, and structured filters.

### Path parameters

<ParamField path="id" type="string" required>UUID of the macro table.</ParamField>

### Query parameters

<ParamField query="page" type="number" default="1">Page number.</ParamField>
<ParamField query="limit" type="number" default="50">Results per page. Clamped between `1` and `200`.</ParamField>
<ParamField query="q" type="string">Full-text search across all display columns.</ParamField>
<ParamField query="obraId" type="string">Filter rows to only those from the obra with this UUID.</ParamField>
<ParamField query="filters" type="string">URL-encoded JSON object of structured filters keyed by column ID.</ParamField>

### Response

<ResponseField name="rows" type="MacroTableRow[]" required>
  Paginated array of row objects. Each row contains:
  - `id` — UUID of the source row from `obra_tabla_rows`
  - `_sourceTablaId` — UUID of the source tabla
  - `_sourceTablaName` — Name of the source tabla
  - `_obraId` — UUID of the parent obra
  - `_obraName` — Name of the parent obra
  - One key per column ID with the resolved value
</ResponseField>

<ResponseField name="columns" type="Column[]" required>Column definitions for the macro table.</ResponseField>

<ResponseField name="pagination" type="object" required>
  <Expandable title="Pagination properties">
    <ResponseField name="page" type="number" />
    <ResponseField name="limit" type="number" />
    <ResponseField name="total" type="number" />
    <ResponseField name="totalPages" type="number" />
    <ResponseField name="hasNextPage" type="boolean" />
    <ResponseField name="hasPreviousPage" type="boolean" />
  </Expandable>
</ResponseField>

        }

        { "id": "col-1-uuid", "label": "Material", "columnType": "source", "dataType": "text", "position": 0 },
        { "id": "col-2-uuid", "label": "Cantidad", "columnType": "source", "dataType": "number", "position": 1 }
      ],
      "pagination": { "page": 1, "limit": 25, "total": 1, "totalPages": 1, "hasNextPage": false, "hasPreviousPage": false }
    }

## Save custom values

<ParamField path="POST /api/macro-tables/{id}/rows" type="string" />

Upserts custom column values for one or more rows. Only columns with `columnType: "custom"` are accepted; values for other column types are silently ignored.

### Path parameters

<ParamField path="id" type="string" required>UUID of the macro table.</ParamField>

### Request body

<ParamField body="customValues" type="CustomValueInput[]" required>
  Array of custom values to upsert.
  <Expandable title="CustomValueInput fields">
    <ParamField body="sourceRowId" type="string" required>UUID of the `obra_tabla_rows` row this value belongs to.</ParamField>
    <ParamField body="columnId" type="string" required>UUID of the custom macro table column.</ParamField>
    <ParamField body="value" type="any" required>The value to store (any JSON-serializable type).</ParamField>
  </Expandable>
</ParamField>

### Response

<ResponseField name="ok" type="boolean" required>Always `true` on success.</ResponseField>
<ResponseField name="updated" type="number" required>Number of custom values upserted.</ResponseField>

<Tabs>
  <Tab title="Request">
    ```bash
    curl -X POST 'https://<domain>/api/macro-tables/mt-uuid-.../rows' \
      --cookie 'sb-access-token=<session>' \
      --header 'Content-Type: application/json' \
      --data '{
        "customValues": [
          {
            "sourceRowId": "row-uuid-...",
            "columnId": "col-custom-uuid",
            "value": "Aprobado"
          }
        ]
      }'
    ```
  </Tab>
  <Tab title="Response">
    ```json
    { "ok": true, "updated": 1 }
    ```
  </Tab>
</Tabs>

### Request body

<ParamField body="roleIds" type="string[]" required>
  Array of role UUIDs that should have this macro table visible in their sidebar.
</ParamField>

### Response

<ResponseField name="ok" type="boolean" required>Always `true` on success.</ResponseField>
